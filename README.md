# 기상청 전자민원 해상 기상특보 수집 · CSV 변환

기상청 전자민원(minwon.kma.go.kr)에서 해상 기상특보(태풍주의보·풍랑주의보) 과거
자료를 연도별로 신청하고 PDF까지 자동 저장한 뒤, 분석용 CSV로 변환하는 스크립트입니다.

한 번에 신청 가능한 용량 제한 때문에 구역을 나눠 반복 신청해야 하는데, 그 반복을
자동화했습니다. 신청 직후 발급 PDF를 규칙에 맞는 폴더/파일명으로 내려받습니다.

```
① 수집  kma_minwon_auto.py / _split.py   →  연도\해역\바다\ 폴더에 PDF
② 변환  pdf_to_csv_batch.py              →  연도별 CSV 1개씩
```

**수집만 할 거면 아래 "수집" 부분만, 이미 받아둔 PDF를 CSV로 바꿀 거면
[CSV 변환](#csv-변환)만 보면 됩니다.**

---

# 수집

## 수집 스크립트는 딱 두 개

| 파일 | 언제 쓰나 | 바꾸는 것 |
|---|---|---|
| `kma_minwon_auto.py` | **개편이 안 걸친 모든 연도** (대부분) | `YEARS`, `ZONE_GROUPS` |
| `kma_minwon_auto_split.py` | **개편이 걸친 연도의 먼바다 보완** (2021·2019·2014만) | `YEARS`, `PERIOD_END`, `FILE_TAG`, `ZONE_GROUPS` |

두 파일의 실행 로직은 완전히 동일합니다. 상단 설정 값만 다릅니다.
아래 블록을 복사해 해당 파일 상단에 붙여넣으면 원하는 연도로 동작합니다.

> **왜 파일이 두 개인가?** 해상 "먼바다" 구역명이 과거 세 번(2014.12.10 / 2019.04.30 /
> 2021.07.29) 개편됐습니다. 개편일이 한 해에 걸치면 그 해 먼바다 자료가 개편일을
> 경계로 두 조각으로 나뉘어, 옛명/현행명을 기간을 쪼개 각각 신청해야 온전한 1년이
> 됩니다. 그 "개편 전 조각"을 받는 게 `_split` 파일입니다. 앞바다는 개편 무관이라
> 항상 기본 파일로 1년 통짜입니다.

## 처음 한 번 세팅

```
conda create -n kma python=3.11 -y
conda activate kma
pip install playwright
```

작업 폴더에 두 스크립트를 두고, 스크립트 상단 `DOWNLOAD_ROOT`(PDF 저장 위치)와
`BOARD_URL`(민원보관함 주소)을 환경에 맞게 확인하세요.

## 매번 실행 절차

1. 엣지를 전부 닫고(작업관리자에서 `msedge.exe` 잔여 프로세스까지 종료), 새 cmd에서:
   ```
   "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --remote-debugging-port=9222 --user-data-dir="C:\edge_automation"
   ```
   경로 오류 시 ` (x86)`를 뺀 경로로 시도.
2. 뜬 엣지 창에서 전자민원 접속 → **PASS 로그인**. 이 창은 작업 끝까지 건드리지 않음.
3. 다른 프롬프트에서:
   ```
   conda activate kma
   python kma_minwon_auto.py        (또는 kma_minwon_auto_split.py)
   ```

## MODE (두 파일 공통)

- `dump` : 사이트의 구역 체크박스 이름 전체 출력 (ZONE_GROUPS 검수용)
- `dry`  : 확인 페이지까지 리허설, 신청/다운로드 안 함 (실전 전 필수 확인)
- `run`  : 실제 신청 + PDF 자동 다운로드

새 연도를 처음 돌릴 땐 **dry로 검증 통과 → 1건만 run 테스트 → 전체 run** 순서를 권장합니다.

## B. 개편 걸친 연도 → `kma_minwon_auto_split.py`

이 파일은 해당 연도의 **먼바다 "개편 전 조각"만** 신청합니다. 그 연도의 앞바다와,
개편과 무관한 다른 해역 먼바다는 A(기본 파일)로 따로 수집해야 합니다.

각 블록의 `YEARS`, `PERIOD_END`, `FILE_TAG`, `ZONE_GROUPS`를 통째로 붙여넣으세요.

### B-1. 2021 (개편일 7.29, 남해서부 제외 6개 해역)

```python
YEARS = [2021]
PERIOD_END = "0729"
FILE_TAG   = "_0101-0729"

ZONE_GROUPS = {
    ("동해중부", "먼바다"): ["동해중부먼바다(~'21.07.29)"],
    ("동해남부", "먼바다"): ["동해남부북쪽먼바다(~'21.07.29)", "동해남부남쪽먼바다(~'21.07.29)"],
    ("서해중부", "먼바다"): ["서해중부먼바다(~'21.07.29)"],
    ("서해남부", "먼바다"): ["서해남부북쪽먼바다(~'21.07.29)", "서해남부남쪽먼바다(~'21.07.29)"],
    ("남해동부", "먼바다"): ["남해동부먼바다(~'21.07.29)"],
    ("제주도", "먼바다"): ["제주도남쪽먼바다(~'21.07.29)"],
}
```

### B-2. 2019 (개편일 4.30, 동해남부·서해남부만)

```python
YEARS = [2019]
PERIOD_END = "0430"
FILE_TAG   = "_0101-0430"

ZONE_GROUPS = {
    ("동해남부", "먼바다"): ["동해남부먼바다(~'19.04.30)"],
    ("서해남부", "먼바다"): ["서해남부먼바다(~'19.04.30)"],
}
```

### B-3. 2014 (개편일 12.10, 남해서부만)

```python
YEARS = [2014]
PERIOD_END = "1210"
FILE_TAG   = "_0101-1210"

ZONE_GROUPS = {
    ("남해서부", "먼바다"): ["남해서부먼바다(~'14.12.10)"],
}
```

> **2014 특이점**: 개편일이 연말이라 "개편 전 조각"이 1.1~12.10로 거의 1년, "개편 후
> 조각"이 12.11~12.31로 약 3주입니다. 분량이 비대칭이지만 정상입니다.

---

## 개편 걸친 연도의 나머지 절반(하반기 조각) 처리

`_split`으로 받은 건 "개편 전" 조각입니다. "개편 후" 조각은 기본 파일(A)이 그 연도를
1년 통짜로 돌릴 때 함께 받아지는데, 이때 파일명에 꼬리표가 없으므로 상·하반기 짝을
맞추려면 하반기 쪽에 꼬리표를 따로 붙여야 합니다.

- 2021 하반기: `_0730-1231`
- 2019 하반기: `_0501-1231` (동해남부·서해남부 먼바다에만)
- 2014 하반기: `_1211-1231` (남해서부 먼바다에만)

2019·2014는 같은 연도의 다른 파일엔 꼬리표가 없어야 정상이라, 일괄 rename이 아니라
**해당 해역 먼바다 파일만 수동으로** 붙이는 걸 권장합니다.

---

## 저장 규칙

```
폴더:   {DOWNLOAD_ROOT}\{연도}\{해역}\{앞바다|먼바다}\
파일명: 기상특보(해상)_{연도}_{해역}_{구분}_{특보}[_기간꼬리표].pdf
```

- 폴더명은 `(1)(2)` 꼬리를 뗀 통일형, 파일명은 `(1)(2)` 유지
  (서해남부 앞바다(1)(2)는 같은 `앞바다` 폴더에 공존, 파일명으로 구분)
- 기간 꼬리표는 개편 걸친 연도의 먼바다에만 붙음. 그 외엔 꼬리표 없는 1년 통짜.

---

## 주의사항

- **개편 걸친 연도(2021·2019·2014)를 기본 파일로 돌리면 안 됨** — 먼바다가 반쪽만
  나옵니다. 반드시 기본(앞바다+무관 해역) + `_split`(개편 해역 먼바다) 조합으로.
- 발급은 신청 후 7일까지만 가능 → 신청한 연도는 바로 다운로드까지 완료(스크립트가
  신청 직후 자동 다운로드하므로 보통 문제 없음).
- 정부 사이트 세션은 약 30분 후 만료될 수 있음 → 연도 단위로 나눠 실행 권장.
- 신청 사이 간격(`DELAY_SEC`, 기본 4초)은 서버 부하 방지용 → 줄이지 말 것.
- 신청 이력은 `신청로그.csv`에 자동 기록됩니다.

# CSV 변환

수집한 PDF는 사람이 읽는 증명서 형식이라 그대로는 분석에 쓸 수 없습니다.
`pdf_to_csv_batch.py`가 폴더를 순회하며 PDF의 특보 표를 추출해 **연도별 CSV 한 개**로
합칩니다.

```
수집(kma_minwon_auto*.py) → PDF → 변환(pdf_to_csv_batch.py) → 연도별 CSV
```

## 세팅

```
conda activate kma
pip install pdfplumber
```

브라우저·로그인은 필요 없습니다. 이미 받아둔 PDF만 읽는 로컬 작업이라 엣지 디버깅
모드도 켜지 않아도 됩니다.

## 실행

```
python pdf_to_csv_batch.py
```

| 설정 | 의미 |
|---|---|
| `DOWNLOAD_ROOT` | PDF가 있는 루트 (수집 스크립트의 `DOWNLOAD_ROOT`와 동일) |
| `OUT_DIR` | CSV 저장 폴더 (기본 `C:\kma_auto\CSV변환`) |
| `ONLY_YEARS` | 처리할 연도 리스트. `[]`이면 전체. 예: `[2023, 2024]` |
| `SKIP_SPLIT_YEARS` | `True`면 기간 꼬리표(`_0101-0729` 등) 붙은 파일을 건너뜀 |

출력: `{OUT_DIR}\기상특보(해상)_{연도}.csv` (인코딩 UTF-8 BOM, 엑셀에서 바로 열림)

> **`SKIP_SPLIT_YEARS`는 보통 `False`로 두세요.** 개편 걸친 연도(2021·2019·2014)는
> 하반기 파일에도 꼬리표를 붙이므로, `True`면 그 연도가 통째로 스킵되어 CSV가 아예
> 생성되지 않습니다. `False`로 두면 상·하반기 파일이 같은 연도 CSV에 이어붙는데,
> 구역명과 발표 시각이 개편일 기준으로 갈리므로 **행이 중복되지 않습니다.**

전체 연도(약 900개 파일)는 30분~1시간 걸립니다. 연도를 몇 개씩 끊어 돌리는 편이
안전합니다. CSV가 엑셀에 열려 있으면 `_잠김대체.csv`로 저장하고 계속 진행합니다.

## 출력 칼럼

| # | 칼럼명 | 설명 | 출처 |
|---|---|---|---|
| 1 | 기상종류 | `기상특보(해상)` 고정 | 고정 |
| 2 | 연도 | 4자리 연도 | 파일명 |
| 3 | 해역 | 7개 해역 | 파일명 |
| 4 | 바다 | 앞바다 / 먼바다 | 파일명 |
| 5 | 세부지역(구역) | 특보가 실제 발표된 예보구역 | PDF 표 |
| 6 | 특보종류 | `풍랑주의보`, `풍랑경보`, `태풍주의보`, `태풍경보` | PDF 표 |
| 7 | 발표_발표 | 특보를 발표한 시각 | PDF 표 |
| 8 | 발표_발효 | 효력이 시작된 시각 | PDF 표 |
| 9 | 변경해제_발표 | 해제를 발표한 시각 | PDF 표 |
| 10 | 변경해제_발효 | 해제 효력 시작 시각 | PDF 표 |
| 11 | 변경해제_내용 | 다른 특보로 전환된 경우의 설명 | PDF 표 |

칼럼 1~4는 파일명에서, 5~11은 PDF 표에서 가져옵니다.

**서해남부 앞바다(1)(2)는 발급 단위일 뿐**이라 CSV에서는 둘 다 `앞바다`로 통합됩니다.
실제 지역 구분은 `세부지역(구역)`에서 됩니다. (두 파일 모두 누락 없이 읽힙니다.)

## 9~11번 칼럼 읽는 법 (중요)

특보가 끝나는 방식에 따라 **두 가지 형태**로 나뉩니다. 이 데이터에서 가장 헷갈리는
부분입니다.

**유형 A — 정상 해제**: 9·10번에 시각, 11번은 빈칸

```
풍랑주의보  2021-01-01 06:00  2021-01-01 08:00  2021-01-02 11:00  2021-01-02 13:00  (빈칸)
```

**유형 B — 다른 특보로 전환**: 9·10번은 빈칸, 11번에 문구

```
풍랑주의보  2021-01-13 11:00  2021-01-13 13:00  (빈칸)  (빈칸)  풍랑경보로 변경
풍랑경보    2021-01-14 10:00  2021-01-14 11:00  (빈칸)  (빈칸)  풍랑주의보로 변경
```

원본 PDF에서 이 칸이 좌우 병합되어 문구 하나만 들어가기 때문입니다.
**유형 B는 종료 시각을 알려면 같은 구역의 다음 행 `발표_발효`를 봐야 합니다.**
지속시간을 계산할 때 이걸 놓치면 조용히 틀립니다.

11번에 등장하는 값: `풍랑경보로 변경`, `풍랑주의보 연장`, `태풍경보로 대치` 등.
**"변경"과 "대치"는 같은 뜻**이며, 원본 PDF 표기가 연도별로 달라 그대로 옮겼습니다.

## 알아둘 것

- **0행 PDF는 정상**입니다. 그 구역·기간에 해당 특보가 없었던 경우입니다.
  (예: 2021년 동해중부·서해중부 태풍 0건)
- **파일명의 특보명 ≠ 실제 특보종류.** 파일명의 `태풍주의보`는 신청 카테고리이고,
  발급 자료엔 "주의보 및 경보"가 함께 들어 있어 `특보종류` 칼럼에는 주의보와 경보가
  섞여 나옵니다. 실제 등급은 **6번 칼럼**을 보세요.
- **개편 걸친 연도는 한 CSV에 옛 구역명과 현행 구역명이 공존**합니다. 중복이 아니라
  시기가 다른 것입니다. 2021년 동해중부 먼바다 예:

  | 세부지역(구역) | 해당 기간 |
  |---|---|
  | `동해중부먼바다` | 1.1 ~ 7.29 (개편 전) |
  | `동해중부안쪽먼바다`, `동해중부바깥먼바다` | 7.30 ~ 12.31 (개편 후) |

  구역명을 인위적으로 통일하지 않고 **그 시점에 유효했던 이름 그대로** 두었습니다.
- 구역명은 상위 접두어를 제거해 저장합니다.
  `동해중부전해상 동해중부바깥먼바다` → `동해중부바깥먼바다`
  `남해동부앞바다 거제시동부앞바다` → `거제시동부앞바다`
- 시각은 모두 KST, `YYYY-MM-DD HH:MM` 문자열입니다. 30분 단위도 존재합니다.
- 정렬은 폴더 순회 순서(해역 가나다순)입니다. 시간순이 필요하면 분석 단계에서
  직접 정렬하세요.
- 폴더명이 `앞바다(1)`처럼 잘못 남아 있어도 **파일명 기준으로 처리**하므로 결과는
  정상입니다. 이 경우 `[참고]` 메시지가 출력되지만 무시해도 됩니다.
